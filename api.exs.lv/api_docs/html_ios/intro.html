Pieprasījuma formāts

Exs pusē visi pieprasījumi no iOS lietotnes tiek gaidīti uz adresi https://ios.exs.lv.
Atbilde uz pieprasījumu vienmēr būs JSON objekts tādā formātā, kāds parādīts zemāk.

json
{
    "success": bool,
    "message": string,
    "ban_type": int,
    "logged_in": bool,
    "xsrf_token": string,
    "response": null|JSONObject
}

Atbildes parametri skaidroti tabulā.

Parametrs Vērtība Skaidrojums
success false ja kaut kas pieprasījuma apstrādē nav izdevies
true veiksmīgas pieprasījuma apstrādes gadījumā
message pie success:false vienmēr satur kļūdas tekstu
pie success:true dažreiz satur informatīvu paziņojumu
(attiecīgajos gadījumos tas pieminēts dokumentācijā)
ban_type 0 pieprasītājam nav piemērots liegums
1 pieprasītāja IP adresei piemērots IP liegums
2 lietotājam piemērots profila liegums
logged_in false lietotājs nav autentificēts
true lietotājs ir autentificēts
xsrf_token simbolu virkne garumā 10, kas jāpievieno turpmāko pieprasījumu galā
(https://ios.exs.lv/..?xsrf=..)
(pārbaudīta gan tiek tikai pie kritiskām darbībām)
response null kad nav jāatgriež papildu saturs, notikusi kļūda, kas konstatējama no iepriekšējiem parametriem, u.tml. situācijas
JSONObject papildu saturs, kas specifisks katram pieprasījumam un aprakstīts citviet

Parametrā success saņemot vērtību "false", parametrs response vienmēr būs vienāds ar "null". success parametrs ir pirmā vērtība, kas lietotnē vienmēr būtu jāpārbauda. Parametrā message esošā vērtība ir paredzēta parādīšanai lietotājam.

Parametrā ban_type saņemot vērtību 1 vai 2, parametrs response var saturēt papildu informāciju, kas aprakstīta sadaļā "Liegumu pārbaude".

Liegumu pārbaude

Katram pieprasījumam tiek veikta IP lieguma pārbaude, savukārt autentificētiem lietotājiem - arī profila lieguma pārbaude.

Konstatējot aktīvu liegumu, oriģinālais pieprasījums vairs netiek ņemts vērā, bet tiek atgriezta atbilde, kurā vēl bez ban_type parametra arī response parametrā tiek pievienota papildinformācija.

IP liegums

IP liegums tiek pārbaudīts katrā pieprasījumā, un pie tāda esamības tiek atgriezta atbilde šādā formātā:

json
{
    "success": true,
    "message": "",
    "ban_type": 1,
    "logged_in": false,
    "xsrf_token": string,
    "response": {
        "ip": string,
        "to_user": {     
            "nick": string,
            "params": string
        },
        "reason": string,
        "from_user": {
            "nick": string,
            "params": string
        },
        "date_from": string,
        "date_to": string,
        "remaining": string
    }
}

Ņemt vērā, ka IP lieguma gadījumā lietotājs tiek automātiski izlogots!

Atbildes parametri skaidroti tabulā.

Parametrs Skaidrojums
ip IP adrese, kurai uzlikts liegums
to_user objekts ar datiem par lietotāju, kuram šāds IP liegums uzlikts
to_user : nick lietotāja lietotājvārds
to_user : params vairākas vērtības, atdalītas ar "|", kas raksturo lietotājvārdu, lai to varētu pareizi noformatēt, piemēram, "115|2|1|3|1":
1) lietotāja ID
2) lietotāja klase: 0 - parasts lietotājs, 1 - administrators, 2 - moderators, 3 - rakstu autors, 5 - bots
3) 0 - lietotājs nav online, 1 - lietotājs ir online
4) ierīce, caur kuru lietotājs skatās lapu: 0 - exs.lv · 1 - m.exs.lv. · 2 - android app · 3 - iOS app
5) vai lietotājam ir aktīvs liegums? 0 - nav, 1 - ir
reason lieguma iemesls
from_user objekts ar datiem par lietotāju, kurš liegumu piemērojis
date_from datums, kad liegums uzlikts (piemēram, "2016-06-29 18:48:12")
date_to datums, kad liegums tiks noņemts (piemēram, "2016-06-30 18:48:12")
remaining lieguma atlikušais laiks (piemēram, "21d 3h 19m 29s")

Profila liegums

Profila liegums tiek pārbaudīts tikai autentificētiem lietotājiem, un atbilde uz pieprasījumu ir identiska tai, kāda tā ir IP lieguma gadījumā ar vienīgo atšķirību parametrā ban_type.

Lieguma datu atsvaidzināšana

Pieprasījums uz adresi https://ios.exs.lv/ban_details ir vienīgais, kas pieejams jebkurā brīdī un ignorē aktīva IP lieguma pārbaudi. Tāds var noderēt, ja lietotnē lietotājam redzams lieguma skats, bet lietotājs ar refresh pogu vēlas datus atjaunot, tā pārbaudot, vai liegums vēl ir spēkā.

Atbilde aktīva lieguma gadījumā ir tāda pati, kā aprakstīts iepriekš, savukārt, ja lieguma nav, atbildei tiek pievienots informatīvs paziņojums:

json
{
    "success": true,
    "message": "Lietotājam nav liegta piekļuve exs.lv.",
    ...
}

Autentifikācijas pārbaude

Pieprasījuma veicējs tiek atpazīts pēc pieprasījumam klāt pievienotā sesijas cepuma. Balstoties uz sesijas cepuma vērtību, parametrā logged_in tiek norādīts, vai šāds lietotājs ir autentificēts. Cepumu nesaņemot, tiek uzskatīts, ka lietotājs nav autentificēts.

Neautentificētam lietotājam ļauts veikt pieprasījumu tikai uz šādām adresēm:

  • https://ios.exs.lv/ban_details
  • https://ios.exs.lv/letmein

Pieprasīt informāciju par lieguma statusu iespējams vienmēr, savukārt autentificēšanās nebūs pieejama, ja pieprasītājam tiks konstatēts IP liegums, par ko attiecīgi tiks pievienota informācija.

Autentificēts lietotājs drīkst veikt pieprasījumu arī uz citām adresēm, kas aprakstītas dokumentācijā.

Autentificēšanās

Autentificēšanās pieprasījums ar POST metodi sūtāms uz adresi https://ios.exs.lv/letmein. Pieprasījumam klāt jāpievieno tabulā norādītās POST vērtības.

Parametrs Vērtības tips
username string
password string

1. Autentificēšanās pieprasījums nebūs pieejams, ja pieprasījuma veicējam tiks konstatēts IP liegums, par ko informācija tiks attiecīgi pievienota atbildē, parametru response saglabājot tukšu ("null").

2. Pēc veiksmīgas autentificēšanās konstatējot, ka lietotājam piemērots profila liegums, tiks pievienota attiecīga informācija, parametru response saglabājot tukšu ("null").

3. Profila liegumam neesot, tiks saņemta šāda atbilde:

json
{
    ...,
    "logged_in": true,
    ...,
    "response": {
        "profile": {
            "nick": string,
            "params": string,
            "avatar_url": string
        }
    }
}

Atbildes parametri skaidroti tabulā.

Parametrs Skaidrojums
nick lietotāja lietotājvārds
params ...
avatar_url saite uz lietotāja avataru

Iziešana

Iziešanas pieprasījums sūtāms uz adresi https://ios.exs.lv/letmeout.

Pieprasījums nebūs pieejams, ja pieprasījuma veicējam tiks konstatēts IP liegums, par ko informācija tiks attiecīgi pievienota atbildē, parametru response saglabājot tukšu ("null").

Pēc lietotāja izautorizēšanas atpakaļ tiks sūtīta šāda atbilde:

json
{
    "success": true,
    "message": "Lietotājs no sistēmas izautorizēts.",
    ...,
    "logged_in": false,
    ...,
    "response": null
}

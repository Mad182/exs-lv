Pieprasījuma un atbildes formāts

Exs pusē visi pieprasījumi no iOS lietotnes tiek gaidīti uz adresi https://ios.exs.lv.

Atbilde uz pieprasījumu ir JSON objekts, kura pirmā atslēga vienmēr ir "status", kas satur skaitlisku vērtību un atspoguļo to, kāds izskatās saturs objektā zem atslēgas "response". Paraugs zemāk:

json
{
    "status": 200,
    "response": {
        "info_message": "Hello world!"
    }
}

Atbilžu veidi

200 - veiksmīgs pieprasījums

Šāda atbilde tiek atgriezta visās situācijās, kad nav jāatgriež kāds no tālāk aprakstītajiem statusiem. Statuss nozīmē, ka atgrieztā informācija atbilst tai, kāda tikusi pieprasīta, un pieprasījumā nav konstatētas kļūdas.

Katrai šāda veida atbildei "response" saturs atšķiras, tāpēc šeit sīkāk nav aprakstīts.

json
{
    "status": 200,
    "response": {}
}

400 - kļūdains pieprasījums

Klienta veiktajā pieprasījumā konstatēta kāda kļūda vai pieprasīto informāciju nevar atgriezt. Parametrā "error_message" vienmēr būs ierakstīts kļūdas iemesls, kādu drīkst parādīt lietotājam.

json
{
    "status": 400,
    "response": {
        "error_message": "Lietotājam nav piekļuves grupai."
    }
}

440 - pieprasījumam jāautentificējas

Ja lietotājs nav autentificējies, bet tiek pieprasīts saturs, kuram tas nepieciešams, tiek atgriezta šāda atbilde:

json
{
    "status": 440,
    "response": {
        "info_message": "Lai piekļūtu saturam, nepieciešams autentificēties."
    }
}

Autentificēšanās sīkāk aprakstīta šajā sadaļā.


441 - pieprasījumam jāveic 2fa autentificēšanās

Atsevišķiem profiliem ir iespējota divu soļu autentificēšanās. Saņemot zemāk parādīto atbildi, jāsaprot, ka lietotājs ar lietotājvārdu un paroli jau ir autentificēts, bet autentificēšanās nav pabeigta un vēl jāiesūta 2fa kods no Google Authenticator.

json
{
    "status": 441,
    "response": {
        "info_message": "Lai pabeigtu autentificēšanos, jāiesūta 2fa kods.",
        "token": "xxxxxxxxxx",
        "profile": {
            "nick": string,
            "params": string,
            "avatar_url": string
        }
    }
}

Parametrs "token" satur simbolu virkni, kas jāpievieno adresei, iesūtot 2fa kodu.
Divu soļu autentificēšanās sīkāk aprakstīta šajā sadaļā.



442 - IP adresei vai autentificētajam profilam ir liegums

Šādā situācijā tiek pievienota "response" atslēga ar JSON objektu, kas sniedz detalizētāku informāciju par liegumu. Šis JSON objekts aprakstīts sadaļā "Liegumu pārbaude".

json
{
    "status": 442,
    "response": {}
}

IP lieguma kontrolei ir augsta prioritāte, kas nozīmē, ka IP lieguma gadījumā šāda atbilde tiks atgriezta pat tad, ja lietotājs centīsies autentificēties.

Liegumu kontrole

Pie katra pieprasījuma tiek veikta liegumu kontrole.

Konstatējot aktīvu liegumu, oriģinālais pieprasījums vairs netiek ņemts vērā, bet tiek atgriezta atbilde ar statusu 442 un JSON objektu atslēgā "response". "response" vērtība satur lieguma informāciju.

JSON atbildes formāts gan IP, gan profila lieguma gadījumā ir identisks.

json
{
    "status": 442,
    "response": {
        "type": int,
        "ip": string,
        "to_user": {     
            "nick": string,
            "params": string
        },
        "reason": string,
        "from_user": {
            "nick": string,
            "params": string
        },
        "date_from": string,
        "date_to": string,
        "remaining": string
    }
}

Ņemt vērā, ka IP lieguma gadījumā lietotājs tiek automātiski izlogots!

Atbildes parametri skaidroti tabulā.

Parametrs Skaidrojums
type lieguma veids: 0 - IP, 1 - profila
ip IP adrese, kurai uzlikts liegums
to_user objekts ar datiem par lietotāju, kuram šāds IP liegums uzlikts
to_user : nick lietotāja lietotājvārds
to_user : params vairākas vērtības, atdalītas ar "|", kas raksturo lietotājvārdu, lai to varētu pareizi noformatēt, piemēram, "115|2|1|3|1":
1) lietotāja ID
2) lietotāja klase: 0 - parasts lietotājs, 1 - administrators, 2 - moderators, 3 - rakstu autors, 5 - bots
3) 0 - lietotājs nav online, 1 - lietotājs ir online
4) ierīce, caur kuru lietotājs skatās lapu: 0 - exs.lv · 1 - m.exs.lv. · 2 - android app · 3 - iOS app
5) vai lietotājam ir aktīvs liegums? 0 - nav, 1 - ir
reason lieguma iemesls
from_user objekts ar datiem par lietotāju, kurš liegumu piemērojis
date_from datums, kad liegums uzlikts (piemēram, "2016-06-29 18:48:12")
date_to datums, kad liegums tiks noņemts (piemēram, "2016-06-30 18:48:12")
remaining lieguma atlikušais laiks (piemēram, "21d 3h 19m 29s")

Lieguma datu atsvaidzināšana

Jebkurā brīdī iespējams izpildīt pieprasījumu uz adresi https://ios.exs.lv/statuses/ban_details, kas apiet IP lieguma pārbaudi un atgriež informāciju par liegumu, ja tāds ir. Atbildes formāts sakrīt ar augstāk redzamo, ar izņēmumu statusa atslēgā, kuras vērtība šajā gadījumā ir 200, kā arī papildparametru "is_active".

json
{
    "status": 200,
    "response": {
        "is_active": bool,
        ...
    }
}

Pieprasījums var noderēt, ja lietotnē lietotājam redzams lieguma skats, bet lietotājs ar refresh pogu vēlas datus atjaunot, tā pārbaudot, vai liegums vēl ir spēkā.

Aktīvam liegumam neesot, tiek atgriezta šāda atbilde:

json
{
    "status": 200,
    "response": {
        "is_active": false,
        "info_message": "Lietotājam nav liegta piekļuve exs.lv."
    }
}

Sadaļas ar neierobežotu piekļuvi

Pieprasījuma veicējs tiek atpazīts pēc pieprasījumam klāt pievienotā sesijas cepuma. Cepumu nesaņemot, tiek uzskatīts, ka lietotājs nav autentificēts.

Adreses, uz kurām pieprasījumu var veikt neautentificēts lietotājs, ir tikai pāris. Galvenās no tām:

  • https://ios.exs.lv/statuses/ban_details
  • https://ios.exs.lv/auth/

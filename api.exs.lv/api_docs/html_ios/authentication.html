Skaidrojums

Lielākā daļa informācijas caur API pieejama tikai tad, ja lietotājs caur API ir autentificējies. Šajā sadaļā aprakstīti ar to saistītie pieprasījumi, kas veicami uz adresi https://ios.exs.lv/auth/.

Pieprasot informāciju no adreses, kur ir nepieciešama lietotāja autentificēšana, un lietotājam neesot autentificētam, tiks atgriezts vai nu 440, vai 441 statuss, kā aprakstīts sadaļā "Ievads".

Atsevišķiem exs profiliem ir aktivizēta divu soļu autentificēšanās, kas nozīmē, ka tiem jāievada ne vien lietotājvārds un parole, bet arī kods, kas iegūts no Google Authenticator lietotnes, vispirms to piesaistot profilam exs web versijā.

Autentificēšana ar lietotājvārdu un paroli

Pieprasījums ar POST metodi sūtāms uz adresi https://ios.exs.lv/auth/login. Pieprasījumam klāt jāpievieno tabulā norādītās POST vērtības.

Parametrs Vērtības tips
username string
password string

Autentificēšanās pieprasījums nav pieejams, ja pieprasījuma veicējam tiek konstatēts IP liegums. Tādā gadījumā atbilstoši aprakstam sadaļā "Ievads" tiek atgriezts 442 statuss.

Neizdevušās autentificēšanās gadījumā tiek atgriezts 400 statuss, informējot par kļūdu, savukārt, visam noritot veiksmīgi, iespējami divi rezultāti:

1. Profilam nav iespējota 2 soļu autentificēšanās.

Tiek pārbaudīts, vai profilam ir piemērots profila liegums. Ja tā ir, tiek atgriezts 442 statuss. Liegumam neesot, atbildē tiek pievienota informācija par autentificētā lietotāja profilu:

json
{
    "statuss": 200,
    "response": {
        "profile": {
            "nick": string,
            "params": string,
            "avatar_url": string
        }
    }
}

Atbildes parametri skaidroti tabulā.

Parametrs Skaidrojums
nick lietotāja lietotājvārds
params ...
avatar_url saite uz lietotāja avataru

2. Profilam ir iespējota 2 soļu autentificēšanās.

Tiek atgriezts 441 statuss šādā formātā:

{
    "statuss": 441,
    "response": {
        "info_message": "Lai pabeigtu autentificēšanos, jāiesūta 2fa kods.",
        "token": "xxxxxxxxxx",
        "profile": {
            "nick": string,
            "params": string,
            "avatar_url": string
        }
    }
}

Lai pabeigtu lietotāja autentificēšanu, jāveic pieprasījums, kas aprakstīts nodaļā "Autentificēšana ar 2fa kodu".

Autentificēšana ar 2fa kodu

Lai pabeigtu divu soļu autentificēšanos, jāiesūta kods, kas iegūstams Google Authenticator lietotnē. (Profilam jābūt iepriekš sasaistītam ar Google Authenticator caur exs web versiju!)

Pieprasījums ar POST metodi sūtāms uz adresi https://ios.exs.lv/auth/2fa?token={token}. Tam klāt jāpievieno tabulā norādītā POST vērtība.

Parametrs Vērtības tips
2fa_code string

Neobligāts ir POST parametrs "2fa_remember". Pievienojot to pieprasījumam, veiksmīgas autentificēšanās gadījumā ierīce, no kuras pieprasījums veikts, tiks iegaumēta, t.i., kādu laiku no tās ievadīt jaunu kodu netiks pieprasīts.

{token} - vērtība, kas iegūstama pirmajā autentificēšanās solī. To nozaudējot, token atkārtota iegūšana aprakstīta šajā pieprasījumā.

Autentifikācijai neizdodoties, tiek atgriezts 400 statuss, savukārt, ja viss noritējis veiksmīgi, atbildei tiek pievienota informācija par autentificēto lietotāju:

json
{
    "statuss": 200,
    "response": {
        "profile": {
            "nick": string,
            "params": string,
            "avatar_url": string
        }
    }
}

Auth token atkārtota iegūšana

Nozaudējot pie autentificēšanās nolasīto tokenu, kas jāpievieno daudzu adrešu galā, to iespējams nolasīt atkārtoti, veicot pieprasījumu uz adresi https://ios.exs.lv/auth/gettoken.

Atpakaļ tiek atgriezta atbilde šādā formātā:

json
{
    "status": 200,
    "response": {
        "token": "xxxxxxxxxx"
    }
}

Iziešana

Iziešanas pieprasījums sūtāms uz adresi https://ios.exs.lv/auth/logout?token={token}.

{token} - pie autentificēšanās saņemtā vērtība atslēgā "token".

Pēc lietotāja izautorizēšanas atpakaļ tiek sūtīta šāda atbilde:

json
{
    "status": 200,
    "response": {
        "info_message": "Lietotājs no sistēmas izautorizēts."
    }
}
